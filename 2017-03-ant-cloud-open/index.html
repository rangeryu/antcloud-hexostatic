蚂蚁云计算开放之路

如果说对蚂蚁金服有所了解的话，大家都知道我们在2015年的10月份正式宣布了发布蚂蚁金融云，当初是有一个想法，就是助力“互联网+”。到目前为止已经过了一年多的时间了，中间也经历了很多故事。今天利用这次机会跟大家分享一下蚂蚁金服的云计算开放之路。

蚂蚁云计算开放之路

蚂蚁的云计算从一开始是为了解决业务规模快速发展中我们切身碰到的一些实实在在的性能、高可用和安全问题等，然后才发展起来的。在2004年到2006年这个第一阶段，支付宝的基础设施还是采用传统架构，它的交易性能上限是每秒一百笔; 第二阶段2007年到2010年的时候我们发觉传统的架构面临性能瓶颈，转变成分布式架构，性能能够达到每秒钟500笔交易; 在2010年发生了一件事情，当时做双十一大促的时候发觉传统的分布式架构缺乏自动化调度能力，难以满足大促性能要求。据说当时一过零点，交易量直冲1500笔每秒，系统面临设计承载力三倍的压力，最终我们通过了人工调度的方式把大促一天的性能压力顶了下来。从那以后，蚂蚁痛定思痛发展了具有自动调度能力的云计算技术，从而解决了性能弹性压力问题。

目前蚂蚁云计算架构已可轻松支持超过每秒12万笔交易的能力，同时在此基础上发展出了很多的黑科技，包括这两天的议程大家都可能也一块来分享到的OceanBase等技术，都会以云服务的方式来提供。放在今天，我们发觉蚂蚁的云计算技术不但对蚂蚁有用，对我们的生态伙伴同样有用，他们非常需要蚂蚁技术的帮助。传统行业在这两年正好进行业务的互联网+转型，各行各业对这方面技术有着迫切的需求，所以我们把蚂蚁多年间积累的技术通过云服务的方式开放出来。我们希望通过云计算技术的开放起到促进业务创新、服务生态的融合作用。

蚂蚁云计算开放历程和服务群体

蚂蚁云计算在最近1年多的开放也经历了三个阶段。第一阶段，于2015年10月份发布了蚂蚁金融云，它做为蚂蚁技术输出的统一出口，承担了促进金融行业互联网+的技术的使命；第二阶段。2016年5月起打通蚂蚁开放平台，我们叫开放融合。以此为契机服务的对象是中小的企业商户，他们对云计算的需求是非常简洁易用，体验非常好，为此我们做了与开放平台帐户融合工作，让开放平台上的商户和ISV能便捷地使用蚂蚁云，使蚂蚁金融云除了服务金融机构之外，还能够服务小微企业；第三个阶段，2016年10月份又把金融云升级了，蚂蚁跟阿里云进行深度合作-我们称之为蚂云计划，把阿里云的能力和蚂蚁的能力融合在一起，统一可以为金融的行业提供更加优质的金融云的服务。

我们主要服务的对象

首先，蚂蚁云计算服务蚂蚁自己的业务，蚂蚁自己的业务要做创新，需要快速获得资源和技术平台进行业务创新落地。蚂蚁云计算可以帮助蚂蚁更快捷的获得创新能力。其次，我们服务有蚂蚁投资的合作伙伴，主要的主题是通过技术合作促进业务融合创新。第三，我们大面积地服务生态的伙伴，通过提供先进、安全的云服务来致力于连接。

升级后的金融云能力框架

升级后的金融云的能力框架，简单讲分了三层：第一层最下面是包括计算能力、容灾、安全、治理、交付都是符合金融级的要求。这部分能力由阿里云以及非常优秀的金融云来提供。中间这一层包括金融级的数据库、中间件、OceanBase、分布式事物、移动式的服务，大数据平台，以及人工智能和应用服务方面的技术，都是蚂蚁近年研究和开发的新型技术，以云服务的方式开放出来，帮助我们的生态伙伴。最上面一层是跟业务相关的，包括一些金融的能力，包括场景的连接，生态等等，所以说融合起来我们升级后的金融云为行业提供了六大能力，包括说IT基础设施的能力，应用构建的能力，管理、运营、金融服务以及生态，生态的融入的能力。

从某金融公司上云案例说起

接下来我就以一个真实的案例来给大家分享一下蚂蚁云计算开放的成果。这是一个真实的一个初创金融公司利用蚂蚁开放的云计算技术实现快速开业的故事。金融类企业主要有银行保险证券等，大家都知道对这些行业有严格的监管。对于金融信息技术它们有一些共性的要求，比如：需要做到同城的两中心双活、异地的数据复制；其次，创新业务将来的快速的发展需要有灵活弹性的基础技术能力。由于初创公司没有传统技术路线包袱，所以一开始就可以做成弹性的架构，支持业务的快速发展；第三，软件的研发发布要实现自动化，快速的发布；还有就是符合等保三级要求，安全合规；最后还要满足6个月之内满足监管审核验收条件，实现快速开业。这是非常苛刻的一个条件，要求非常高，时间非常紧。

我们为其提供了利用蚂蚁云上的解决方案。第一，提供了蚂蚁的金融级的分布式的框架以及相关的中间件，也就是支持支付宝每秒钟12万笔的整套的框架和组件。第二，提供全套移动开发运营的框架和平台，我们叫mPaaS，移动应用可以直接用这套平台开发，非常快速的搭建出来类似于支付宝钱包这种规模、这种能力，这种可用性、可靠性的一个移动的应用；第三，还采用了金融的大数据平台采云间，以及客服系统采用的是智能云客服，这个客服系统也是支付宝的客服的技术，它是非常有特点的，它的人工智能技术可以满足90%几的响应，大幅度减轻人员的工作压力，提高客服效率；最后，还提供一站式的开发平台云效平台。

我们再分别看一下。金融级的分布式中间件可能大家已经听说过了，这个是蚂蚁的非常重要的一套中间件。分布式大家都听说过，蚂蚁之外也有很多组件，有其它公司自行开发的，也有开源的，但是真正能够满足金融级的交易一致性和数据的可靠性等要求的分布式技术产品，在业界还是非常少的，而能够对外开放的金融级分布式中间件就更少了。蚂蚁因为有十几年的支付宝的这样的经历，本身就是跟金融机构打交道，所以它对数据的一致性、安全是非常重视的，蚂蚁设计的这套开发框架和分布式中间件就是专门针对金融级分布式应用苛刻的非功能性的要求来做的。

云：金融级分布式中间件

它有三大特点，特点一是分布式，分布式提供了高性能和高弹性能力；特点二是金融级，利用这套框架和中间件可以很方便地实现同城双活，且满足数据的一致性、安全性要求。特点三是自动化，分布式之后，未来应用要发布，要对应用进行管理，如果只靠人工来做会根本无法完成，所以需要借助技术，蚂蚁云计算具备一键发布，智能运维的能力，来帮助我们这套系统平滑运转。整套分布式中间件含六大组件，包括分布式的服务注册中心，分布式事务实现一致性，分布式数据源实现数据的横向扩展，有了这三个组件之后，理论上来讲已经能够满足风控和金融级的要求了。利用分布式消息队列通过服务解耦实现性能提升，加上分布式的动态配置和分布式的调度实现自动化，可以管理几千个节点的集群没有压力。再加上配套的运维管理和监控安全等措施。整套产品可以支持银行、保险等金融企业面向互联网+技术升级，实现横向扩展、满足性能弹性要求，同时满足金融行业安全要求。

端：移动研发平台mPaaS

第二个提供的是“端”这一块技术能力，所谓移动研发平台mPaaS。这是一个覆盖了研发、运行和运维三个应用生命周期，同时它既有开发的框架，又有推送、分析、网关等整套能力。后面有专门的议题对它会做比较详细的分析和介绍。我这里只说一点，在国内支付宝钱包是第三大使用频率的一个APP。所谓第三大，也就意味着每天有成千上万，上亿次的访问，所以它面临的使用的环境是千奇百怪，各种各样的，比如说我们现在大家已经习惯4G网络，但是还有很多山沟沟里面只有2G信号，支付宝钱包还是能打开支付。还有我们需要比如说做红包的业务，需要频繁地做迭代更新，mPaaS可以帮你实现补丁热更新。

数：金融大数据平台

第三个是大数据的平台。简单来讲是一套BI，只不过说这套BI可以在云上做的，而不是我们一定要买一套设备，然后放在机房里面，花了好多钱维护，它在云上，起步非常小，几千块钱或者几百块钱一个月可以用起来了，它起步非常的低，你可以快速的发展，成本仍然非常低。

蚂蚁零号云客服，客户服务智能化

还要讲一下云客服，云客服是我们SaaS的服务，把支付宝对几亿的客户的服务的服务中心，把它的技术能力开放出来，服务我们的伙伴。它的最大的一个特点在于它后面有一个智能化，94%的客服请求是由虚拟的客服来解决的，只有6%需要人工介入。它的能力的提升可能是20倍，所以说如果说碰到一个双十一的这样一种场景的话，每天可能有几百万的请求，如果用人工来接的话，整个蚂蚁全部做客服都不够，所以一定要有人工智能的技术来帮助我们提升效率，并且它服务的还是非常好的。这个技术也已经开放出来，在云上面是可以提供的。

一站式研发平台-云效平台

还有一个非常有特点的叫云效平台，可以看到要实现智能化的开发，快速的迭代，一定要有相应的流程、工具支撑它，云效平台可以帮助我们在需求端，开发的时候进行测试、发布，以及最后来复盘它的一个问题，整套的流程都在这个平台管理，大大提升了我们的研发效率。经过以上这几个解决方案的组合形成行业解决方案，在双方几个月来齐心协力，这一家金融客户终于在只有6个月时间之内能够满足监管审核条件，这是非常非常不容易的。并且因为他采用的是公有云方式，其在IT上投入的资金可能意想不到的便宜，它能做到两地三中心，并且是双活的，然后也能够满足这些条件，每个月支出的成本，IT上的投入只有几十万，如果自己建，6个月之内要完成显然是不可能的。通常招个标都要几个月，并且整体投入是亿元级的。

云计算建设的几点启示

通过这段时间蚂蚁云计算开放的工作，还是有一点启示的，我们讲做云计算来讲的话，有几块的心得和大家分享一下：

第一个启示：如果只从IT部门的视角考虑建设云计算，我们通常更多的会考虑成本、效率，但事实上要做云计算的决策或者云计算的业务，更应该从业务价值方面来考虑，咱们做这个云计算到底为业务带来了什么，只有从这方面考虑可能我们的云计算价值才能真正体现。从IT的投入可能节省了一部分，效率可能提升了，但是这不是云计算的全部。而对生态的引入，对于伙伴的支持，使得你的业务得到了快速的发展，原来无法实现的创新，现在在云计算的平台之下可以做了，这个价值远远超过IT投入的成本节省。因此要从业务的价值来思考云计算的价值。

第二个启示：每个做云计算决策或者要使用或者服务的都会碰到这个问题，公有云还是专有云，专有云在金融行业用的比较多的，虚拟化，一开始就虚拟化用的比较多的，甚至还会涉及研发布布的一体化能力，而公有云除了能够提供这些好处之外，还能够给我们带来生态的价值。在专有云的环境里，我们需要要自己建很多东西，可在公有云上，若别人已经提供云服务了，我们引过来用就可以。这里面有所得有所失，我们安全怎么考虑？当然这方面要考虑，另外一方面可以最快速的支撑你，帮助业务实现升级创新。第二个是自建还是外购？那我相信很多大企业也好，大的金融机构也好，自建比较多，但自建看你怎么建，有的从头开始建，实际上是蛮累的，最近几年也接触过一些大的金融机构，实际上真的很辛苦，要自己建从头开始，过两年跟不上要铲掉重来，投入非常大。自建或者外购是一个选择，这不是最重要的，最重要的是我做云计算的价值，是对业务的价值，能够快速达到的话自建不是一个问题，哪种方式可以更快速的达到业务价值，这就是我们要做的。第三个要考虑的是服务内部还是服务生态，蚂蚁的云计算的开放是既要服务内部，又要服务生态。服务内部简单一点，把系统建起来就可以了，但是服务生态还要考虑生态的商业上的一些能力，包括说计费、计量、商业模式，那就更复杂一些。

第三个启示：云计算看上去是一个技术，实际上带来的影响可能是超越技术的。比如说我们可能跟它相关的业务的流程也需要做一些变革。那我们的组织结构甚至也要做变革来帮助云计算的成功或者真正的发挥效益。举个例子，我们现在在服务一些企业的时候，发觉它原来那套体系都是以传统的方式来搭建的，项目推进还需要做招投标，然后签约之后它的付款方式要么就是说分期，首期百分之多少，中间到某个程度百分之多少，还是传统的方式。但实际上云计算的方式不是这样来算的，它的一个特点叫做随需所用，用多少付多少钱，你可以少付钱得到更多的服务，我们可以直接扣款完成交易，这种交易流程的简便提升了交易效率，对双方是双赢的。但是很多企业的传统业务流程还没有调整过来，还是用老的一套，流程较长，人员投入大，致云计算的实际效益大打折扣的。

蚂蚁云计算开放的成果小结

蚂蚁的云计算开放对于蚂蚁金服生态和业务的促进，总结下来有以下四个方面：

创新。有很多是我们蚂蚁自己的业务部门在云计算平台上在跑它的系统。这个好处就是说它可以快速的得到它所需的技术能力，所以说目前来讲在蚂蚁云的平台上入驻已经有行业开放平台，它是支撑各个行业快速的跟支付宝形成支付等等，目前已经有十个行业入驻了，今年可能还要翻倍。由此可以推广开，其它BU也逐渐越来越多以借助蚂蚁云的技术平台进行快速创新。

融合。蚂蚁云计算促进的是蚂蚁生态的融合。我们可以把它打造成一些，除了金融机构，服务小微企业的IT平台，小微企业的IT能力比较弱的，上了这个平台就大大提升技术能力。

安全。安全是云上非常重要的话题，也是蚂蚁云的特色。蚂蚁自己也有很多的数据在这个云平台上，便得在蚂蚁云平台上的一些创新可以做得更加灵活和放心。因为它可以保护大家的数据，尤其云平台上面还有一个叫数据实验室，它是把双方的数据放在一起来做共创，并且是脱敏来共创，不用担心数据丢失，可以共享成果。

第四部分是未来要做的，现在刚刚开始尝试，希望在云上面做成行业的解决方案，针对行业不单单提供蚂蚁的技术，而要引入蚂蚁的业务的能力，一起来做大做强。所以说我们会有四大的能力，可以在蚂蚁上以来实现，帮助我们业务实现价值。
